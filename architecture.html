Technical overview

Vault architecture

Vault is an offline password manager that runs entirely in the browser. Credentials are encrypted locally and stored on the user’s device. No passwords are transmitted to a server and no cloud service is required.

Core model

  • Client-side encryption using the browser’s Web Crypto APIs.
  • Local persistence using browser storage on the device.
  • Manual export and import of the encrypted vault file.
  • Offline capability through PWA installation and a service worker.

Design intent

The design stays deliberately small and understandable: static pages, vanilla JavaScript, local encryption, and no dependency on a remote account system.

Application shape

  • app.html – app shell and runtime entry point.
  • app.js – main application controller.
  • storage.js – local save, load, import, export, and entry IDs.
  • crypto.js – client-side encryption helpers.
  • UI modules – entry rendering, dialogs, icons, search, reset, and auto-lock behaviour.

Security boundaries

Vault reduces dependency on third-party infrastructure, but it does not claim to protect against a compromised device, weak passwords, careless backups, or unsafe operating environments.

Practical summary

Technically, Vault aims for clarity rather than complexity: local encryption, local storage, deliberate backup movement, and a small code surface that can be inspected and maintained.

This page is intended for technical readers who want the trust model and implementation outline in plain language.
